Sale Screen Overview

The layout and design for all screens used for register functions is consistent throughout Oracle Retail Xstore Point of Service .

  1. Information Area Navigation: To view the tabs, use a mouse and click the tab, touch the tab if you are using a touch screen, or use the keyboard [Ctrl]+[Tab] and [Ctrl]+[Shift]+[Tab] to navigate between each tab.

Info tab - Displays product-specific information for an item-entry scenario, customer/item detail information for a layaway transaction, or instructions for recording a repair or alteration.

Tasks tab - Displays open and in progress tasks for the current system day. Tasks shown here are view only.

Store Goals tab - Displays information from the corporate office about the goals set for your store for a specified date range. During a sale transaction, the Goals tab is replaced by the Coupons tab.

Coupons tab - Lists all of the deal coupons scanned during a transaction.

Messages tab - Displays information from the corporate office, or from your store management.

Keypad tab - Displays a numeric keyboard for entering numbers and a keyboard for entering alpha characters.

  1. Focus Bar - The focus bar provides a single-entry field in which only one entry for a specified piece of data may be captured and then stored in the database. The message area in the lower part of the focus bar displays instructions for the current operation. The smaller text area above the entry field shows the title of the focus bar. The focus bar changes color to indicate the specific function you are performing.
  2. View Port - This area of the screen presents all pertinent information about the current function. For example, in a sale transaction, the View Port displays the sale transaction information for items, taxes, discounts, tenders or any other detailed data about the current sales transaction.
  3. Menu Buttons - The Oracle Retail Xstore Point of Service menu buttons show the functions that appear on each Oracle Retail Xstore Point of Service window. Each menu button has a function key (F1 - F12) assigned to it. Because each Oracle Retail Xstore Point of Service menu is configurable, the button functions on your system may appear in a different order or have different text than examples shown here. Some menu buttons may not be available to every associate. Security configuration determines which options are available to an individual associate.
  4. Status Bar - The Status Bar is a screen component that displays several items of store-related information, and is always visible from every Register screen and Back Office screen.

The messaging section of the status bar shows the Messages and Datasource components. To view additional information you must have a touch-screen or a mouse to click the buttons and view additional detail.

Urgency levels are conveyed with stoplight-like color indicators:

  • Red = High Urgency
  • Yellow = Medium Urgency
  • Green = Low Urgency (Informational conditions)
  • The register section of the status bar shows the following information:
  • Store ID – The store identifier (Yellow or red indicates the store is closed, depending upon Oracle Retail Xstore Point of Service version.)
  • Register ID – The ID number assigned to the register (Red indicates the register is closed.)
  • Cashier ID – The employee ID number or name of the associate signed on to the register
  • Register Date/Time – Indicates the date/time as related to POS sales

See also: the Oracle Retail Xstore Point of Service User Guide for more information about Oracle Retail Xstore Point of Service screen conventions.
